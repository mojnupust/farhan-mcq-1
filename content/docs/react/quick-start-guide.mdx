## 1. React কী এবং কেন?

**React** হলো Facebook (Meta)-এর তৈরি একটি **JavaScript UI Library**। এটা দিয়ে আমরা ওয়েব অ্যাপ্লিকেশনের **User Interface (UI)** বানাই।

### React কেন ব্যবহার করবো?

ধরুন আপনি একটি পুরনো পদ্ধতিতে (Plain JavaScript + DOM Manipulation) একটি লিস্ট বানালে — ছাত্র যোগ করলে, মুছলে, আপডেট করলে প্রতিবার পুরো DOM রিফ্রেশ করতে হয়। এটা **ধীর** এবং **ত্রুটিপ্রবণ**।

React-এ **Virtual DOM** আছে। React প্রথমে মেমোরিতে একটি Virtual DOM তৈরি করে, তারপর আসল DOM-এর সাথে তুলনা করে (**Diffing**) — শুধু যা পরিবর্তন হয়েছে সেটুকুই আপডেট করে (**Reconciliation**)। এতে পারফরম্যান্স অনেক ভালো।

### React-এর মূল মানসিকতা (Mental Model)

```
UI = f(state)
```

অর্থাৎ — **UI হলো state-এর একটি ফাংশন।** State পরিবর্তন হলে UI স্বয়ংক্রিয়ভাবে আপডেট হয়। আপনাকে DOM manually আপডেট করতে হবে না।

## 3. Vite দিয়ে প্রজেক্ট তৈরি

### Vite কেন?

পুরনো `create-react-app` অনেক ধীর। **Vite** আধুনিক JavaScript Bundler যা:
- **esbuild** ব্যবহার করে — ১০০x দ্রুত ট্রান্সপাইল করে
- **Hot Module Replacement (HMR)** — কোড সেভ করলেই browser আপডেট হয়, page refresh ছাড়াই
- Production build-এ **Rollup** ব্যবহার করে — optimized output

### প্রজেক্ট তৈরি

```bash
# Vite দিয়ে React প্রজেক্ট তৈরি
npm create vite@latest student-result-tracker -- --template react

# প্রজেক্ট ফোল্ডারে যাও
cd student-result-tracker

# Dependencies ইনস্টল করো
npm install

# Development server চালু করো
npm run dev
```

Browser-এ `http://localhost:5173` খোলো — React অ্যাপ দেখবে।

### Tailwind CSS সেটআপ

আমরা styling-এর জন্য **Tailwind CSS** ব্যবহার করবো — এটি utility-first CSS framework। প্রতিটি class একটি specific CSS property control করে।

```bash
npm install tailwindcss @tailwindcss/vite
```

**`vite.config.js`** আপডেট করো:

```js
import tailwindcss from "@tailwindcss/vite";
import react from "@vitejs/plugin-react";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [react(), tailwindcss()],
});
```

**`src/index.css`** এ Tailwind import করো:

```css
@import "tailwindcss";
```

## 5. JSX — JavaScript-এর ভেতরে HTML

**JSX (JavaScript XML)** হলো JavaScript-এর একটি **syntax extension** — এটা দেখতে HTML-এর মতো কিন্তু আসলে JavaScript। Browser সরাসরি JSX বোঝে না — Vite/Babel এটাকে pure JavaScript-এ রূপান্তরিত করে।

### JSX আসলে কী করে?

```jsx
// আপনি লিখুন:
const element = <h1 className="title">হ্যালো, React!</h1>;

// Babel রূপান্তরিত করে:
const element = React.createElement("h1", { className: "title" }, "হ্যালো, React!");
```

দেখো — JSX মূলত `React.createElement()` এর syntactic sugar।

### JSX-এর নিয়মাবলি

**নিয়ম ১ — একটি Root Element থাকতে হবে:**

```jsx
// ❌ ভুল — দুটো sibling element
return (
  <h1>শিরোনাম</h1>
  <p>প্যারাগ্রাফ</p>
);

// ✅ সঠিক — একটি wrapper div বা Fragment
return (
  <div>
    <h1>শিরোনাম</h1>
    <p>প্যারাগ্রাফ</p>
  </div>
);

// ✅ আরও ভালো — Fragment (extra div যোগ হয় না DOM-এ)
return (
  <>
    <h1>শিরোনাম</h1>
    <p>প্যারাগ্রাফ</p>
  </>
);
```

**নিয়ম ২ — `class` নয়, `className` লিখতে হবে:**

```jsx
// ❌ ভুল
<div class="container">...</div>

// ✅ সঠিক (কারণ 'class' JavaScript-এ reserved keyword)
<div className="container">...</div>
```

**নিয়ম ৩ — JavaScript Expression `{}` দিয়ে:**

```jsx
const studentName = "রাহেলা আক্তার";
const marks = 85;

return (
  <div>
    <h2>{studentName}</h2>
    <p>মার্কস: {marks}</p>
    <p>পাস/ফেল: {marks >= 33 ? "পাস ✅" : "ফেল ❌"}</p>
    <p>গ্রেড: {marks >= 80 ? "A+" : "A"}</p>
  </div>
);
```

> `{}` এর ভেতরে **expression** লেখা যায় কিন্তু **statement** (যেমন `if`, `for`) নয়।

**নিয়ম ৪ — Self-closing Tags:**

```jsx
// ❌ HTML-তে এটা চলে, JSX-এ না
<input type="text">
<br>
<img src="photo.jpg">

// ✅ JSX-এ সবসময় close করতে হবে
<input type="text" />
<br />
<img src="photo.jpg" />
```

### আমাদের প্রজেক্ট: `Summary.jsx`

```jsx
// src/components/Summary.jsx

export default function Summary({ totalStudents, averageMarks }) {
  return (
    <div className="grid grid-cols-2 gap-4 p-4 bg-white rounded-lg shadow">

        <p className="text-3xl font-bold text-green-600">
          {averageMarks.toFixed(1)}
        </p>
      </div>
    </div>
  );
}
```

## 6. Component — UI-এর ছোট ছোট ব্লক

**Component** হলো React-এর সবচেয়ে মূল ধারণা। প্রতিটি Component হলো একটি **পুনর্ব্যবহারযোগ্য (reusable) UI ব্লক** — যেটা নিজের data (state) রাখতে পারে এবং বাইরে থেকে data (props) নিতে পারে।

### Functional Component

আজকের React-এ শুধু **Functional Component** ব্যবহার করা হয়। এটি একটি সাধারণ JavaScript function যা JSX return করে।

```jsx
// একটি সরল Functional Component
function Greeting() {
  return <h1>স্বাগতম, React জগতে! 🎉</h1>;
}

export default Greeting;
```

**কনভেনশন:**
- Component-এর নাম সবসময় **PascalCase** (বড় হাতের প্রথম অক্ষর) — যেমন `StudentCard`, `GradeFilter`
- ফাইলের নামও Component-এর নামের মতো হবে — `StudentCard.jsx`

### Component Tree (কম্পোনেন্ট গাছ)

আমাদের অ্যাপের Component tree এভাবে:

```
App
├── Header
├── Summary
├── GradeFilter
├── AddStudentForm
└── StudentList
    └── StudentCard (একাধিক)
```

Parent component তার child-কে data দেয় **props** দিয়ে।

### আমাদের `App.jsx`


### Class Component (ঐতিহাসিক জ্ঞান)

React-এর পুরনো ভার্সনে **Class Component** ব্যবহার হতো। এখন প্রায় আর দেখা যায় না, তবে পুরনো কোড বুঝতে কাজে আসে।

```jsx

  render() {
    return (

        <h1>Student Result Tracker</h1>

    );
  }
}
```

**Class vs Functional:**

| বিষয় | Class Component | Functional Component |
|------|----------------|---------------------|
| Syntax | `class Foo extends Component` | `function Foo()` |
| State | `this.state` | `useState` Hook |
| Lifecycle | `componentDidMount` ইত্যাদি | `useEffect` Hook |
| জনপ্রিয়তা | পুরনো, কমে আসছে | আধুনিক, সবাই ব্যবহার করে |

> ✅ **সিদ্ধান্ত:** সবসময় **Functional Component** লিখো।

## 7. Props — Parent থেকে Child-এ ডাটা

**Props (Properties)** হলো Component-এ বাইরে থেকে data পাঠানোর উপায়। Props **read-only** — child component props পরিবর্তন করতে পারে না।

### Props কীভাবে কাজ করে?

```jsx

  roll="2024001"
  averageMarks={86.2}

      <h3>{name}</h3>
      <p>রোল: {roll}</p>
```

```jsx
// ❌ পুরনো উপায় — verbose
function StudentCard(props) {

// ✅ আধুনিক উপায় — Destructuring
```

## 8. State ও useState — UI জীবন্ত করা

**State** হলো Component-এর নিজস্ব **মনে রাখার ক্ষমতা**। State পরিবর্তন হলে React স্বয়ংক্রিয়ভাবে UI re-render করে।

### useState Hook

```jsx
import { useState } from "react";

function Counter() {
  // Syntax: const [stateValue, setterFunction] = useState(initialValue);
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>কাউন্ট: {count}</p>
      <button onClick={() => setCount(count + 1)}>বাড়াও ▲</button>
      <button onClick={() => setCount(count - 1)}>কমাও ▼</button>
      <button onClick={() => setCount(0)}>রিসেট</button>
    </div>
  );
}
```

### State-এর গুরুত্বপূর্ণ নিয়মাবলি

**নিয়ম ১ — State directly mutate করবে না:**

```jsx
// ❌ ভুল — React জানবে না state পরিবর্তন হয়েছে
const [students, setStudents] = useState([...]);
students.push(newStudent); // re-render হবে না!

// ✅ সঠিক — নতুন array তৈরি করো
setStudents([...students, newStudent]); // re-render হবে
```

**নিয়ম ২ — পুরনো state-এর উপর নির্ভর করলে functional update ব্যবহার করো:**

```jsx
// ❌ সম্ভাব্য সমস্যা (stale closure)
setCount(count + 1);

// ✅ নিরাপদ (functional update)
setCount((prevCount) => prevCount + 1);
```

**নিয়ম ৩ — Object বা Array state আপডেটে spread operator ব্যবহার করো:**

```jsx
const [student, setStudent] = useState({ name: "রাহেলা", marks: 85 });

// ❌ ভুল — পুরো object replace হয়
setStudent({ marks: 90 }); // name হারিয়ে যাবে!

// ✅ সঠিক — spread করে নতুন object তৈরি
setStudent({ ...student, marks: 90 });
```

### আমাদের প্রজেক্টে State যোগ করা

এখন `App.jsx` আপডেট করো যাতে ছাত্রের তালিকা state-এ রাখা যায় এবং নতুন ছাত্র যোগ করা যায়:

```jsx
// src/App.jsx (updated)
import { useState } from "react";
import Header from "./components/Header";
import Summary from "./components/Summary";
import AddStudentForm from "./components/AddStudentForm";
import StudentList from "./components/StudentList";
import GradeFilter from "./components/GradeFilter";
import { students as initialStudents } from "./data/students";

function getAverage(s) {
  return (s.bangla + s.english + s.math + s.science + s.socialScience) / 5;
}

export default function App() {
  const [students, setStudents] = useState(initialStudents);
  const [filterGrade, setFilterGrade] = useState("all");

  // গড় নম্বর হিসাব
  const overallAverage =
    students.length > 0
      ? students.reduce((sum, s) => sum + getAverage(s), 0) / students.length
      : 0;

  // নতুন ছাত্র যোগ করা
  function handleAddStudent(newStudent) {
    setStudents([...students, { ...newStudent, id: Date.now() }]);
  }

  // ছাত্র মুছে ফেলা
  function handleDeleteStudent(id) {
    setStudents(students.filter((s) => s.id !== id));
  }

  // গ্রেড অনুযায়ী ফিল্টার
  function getGrade(avg) {
    if (avg >= 80) return "A+";
    if (avg >= 70) return "A";
    if (avg >= 60) return "B";
    if (avg >= 50) return "C";
    if (avg >= 33) return "D";
    return "F";
  }

  const filteredStudents =
    filterGrade === "all"
      ? students
      : students.filter((s) => getGrade(getAverage(s)) === filterGrade);

  return (
    <div className="min-h-screen bg-gray-100">
      <Header />
      <main className="max-w-4xl mx-auto p-4 flex flex-col gap-6">
        <Summary
          totalStudents={students.length}
          averageMarks={overallAverage}
        />
        <GradeFilter
          selectedGrade={filterGrade}
          onSelectGrade={setFilterGrade}
        />
        <AddStudentForm onAddStudent={handleAddStudent} />
        <StudentList
          students={filteredStudents}
          onDeleteStudent={handleDeleteStudent}
        />
      </main>
    </div>
  );
}
```

## 9. Events — ইউজারের ইন্টারঅ্যাকশন

### Event Handler-এর নিয়ম

```jsx
function MyButton() {

    alert("বাটনে ক্লিক হয়েছে!");
  }

  // ✅ সঠিক — function reference পাস করা হচ্ছে
  return <button onClick={handleClick}>ক্লিক করো</button>;

  // ❌ ভুল — render-এর সময়ই function call হয়ে যাবে!
  return <button onClick={handleClick()}>ক্লিক করো</button>;
}
```

প্রতিটি event handler একটি **SyntheticEvent** object পায়:

## 10. List ও Keys — অ্যারে থেকে UI

React-এ array-এর প্রতিটি item থেকে UI তৈরি করতে JavaScript-এর `Array.map()` ব্যবহার করা হয়।

### `key` কেন দরকার?

React যখন একটি list render করে, প্রতিটি item-কে **unique key** দিয়ে identify করে। State পরিবর্তন হলে React শুধু সেই item-ই আপডেট করে যার data বদলেছে — পুরো list নয়। এটাই **Reconciliation** এর মূল নীতি।

```jsx
// ❌ key নেই — React সঠিকভাবে track করতে পারবে না
{students.map((s) => <StudentCard student={s} />)}

// ❌ index key — list reorder হলে সমস্যা
{students.map((s, index) => <StudentCard key={index} student={s} />)}

// ✅ unique id key — সবচেয়ে ভালো
{students.map((s) => <StudentCard key={s.id} student={s} />)}
```

> **⚠️ সতর্কতা:** Array index-কে key হিসেবে ব্যবহার করবে না যদি list-এর items add/remove/reorder হতে পারে। এতে subtle bugs হয়।

## 11. Conditional Rendering

React-এ শর্ত অনুযায়ী UI দেখানো বা না দেখানোর জন্য বিভিন্ন পদ্ধতি আছে।

### পদ্ধতি ১ — Ternary Operator

```jsx
// condition ? trueUI : falseUI
{students.length > 0 ? (
  <StudentList students={students} />
) : (
  <p className="text-center text-gray-400 py-10">কোনো ছাত্র নেই 😕</p>
)}
```

### পদ্ধতি ২ — Short-circuit (`&&`)

```jsx
// condition && UI (condition false হলে কিছু দেখাবে না)
{isLoading && <p>লোড হচ্ছে...</p>}
{error && <p className="text-red-500">Error: {error}</p>}
```

> **⚠️ সতর্কতা:** `0 && <Component />` লিখলে `0` render হবে। তাই `!!length && <Component />` অথবা `length > 0 && <Component />` লিখো।

### পদ্ধতি ৩ — Early Return

```jsx
function StudentList({ students }) {
  // Early return — empty state আলাদা render করো
  if (students.length === 0) {
    return (
      <div className="text-center py-10">

        <p className="text-gray-500">এই গ্রেডে কোনো ছাত্র নেই।</p>
      </div>
    );
  }

  return (
    <div className="flex flex-col gap-4">
      {students.map((s) => <StudentCard key={s.id} student={s} />)}
    </div>
  );
}
```

## 12. Forms ও Controlled Components

React-এ form handling-এর সবচেয়ে নির্ভরযোগ্য পদ্ধতি হলো **Controlled Component** — যেখানে form-এর প্রতিটি input-এর value React state-এ রাখা হয়।

### Controlled vs Uncontrolled

| | Controlled | Uncontrolled |
|--|------------|--------------|
| Data রাখে | React state | DOM |
| Access | `state` value দিয়ে | `ref` দিয়ে |
| Validation | সহজ, real-time | কঠিন |
| **ব্যবহার করো** | ✅ সবসময় | শুধু বিশেষ ক্ষেত্রে |

### আমাদের `AddStudentForm.jsx`

```jsx
// src/components/AddStudentForm.jsx
import { useState } from "react";

const initialFormData = {
  name: "",
  roll: "",
  bangla: "",
  english: "",
  math: "",
  science: "",
  socialScience: "",
};

export default function AddStudentForm({ onAddStudent }) {
  const [formData, setFormData] = useState(initialFormData);
  const [errors, setErrors] = useState({});
  const [isOpen, setIsOpen] = useState(false);

  // একটিমাত্র handler দিয়ে সব input পরিচালনা
  function handleChange(event) {
    const { name, value } = event.target;
    setFormData((prev) => ({ ...prev, [name]: value }));
    // টাইপ করার সাথে সাথে error সরিয়ে দাও
    if (errors[name]) {
      setErrors((prev) => ({ ...prev, [name]: "" }));
    }
  }

  function validate() {
    const newErrors = {};
    if (!formData.name.trim()) newErrors.name = "নাম দিতে হবে";
    if (!formData.roll.trim()) newErrors.roll = "রোল নম্বর দিতে হবে";

    const markFields = ["bangla", "english", "math", "science", "socialScience"];
    markFields.forEach((field) => {
      const val = Number(formData[field]);
      if (formData[field] === "") {
        newErrors[field] = "নম্বর দিতে হবে";
      } else if (isNaN(val) || val < 0 || val > 100) {
        newErrors[field] = "০ থেকে ১০০ এর মধ্যে হতে হবে";
      }
    });
    return newErrors;
  }

  function handleSubmit(event) {
    event.preventDefault(); // page reload বন্ধ করো

    const validationErrors = validate();
    if (Object.keys(validationErrors).length > 0) {
      setErrors(validationErrors);
      return;
    }

    onAddStudent({
      ...formData,
      bangla: Number(formData.bangla),
      english: Number(formData.english),
      math: Number(formData.math),
      science: Number(formData.science),
      socialScience: Number(formData.socialScience),
    });

    setFormData(initialFormData); // form reset
    setIsOpen(false);
  }

  if (!isOpen) {
    return (
      <button
        onClick={() => setIsOpen(true)}
```

## 13. useEffect — Side Effects ও API Call

**`useEffect`** হলো React-এর একটি Hook যা দিয়ে **side effects** পরিচালনা করা হয়। Side effect মানে হলো — এমন কাজ যা Component render করার বাইরে হয়: API call, localStorage, subscription, timer ইত্যাদি।

### useEffect-এর Syntax

```jsx
useEffect(() => {
  // ১. এখানে side effect কোড লেখো

  return () => {
    // ২. Cleanup function (optional)
    //    Component unmount হওয়ার আগে বা
    //    effect পুনরায় চলার আগে এটা call হয়
  };
}, [dependency1, dependency2]); // ৩. Dependency array
```

### Dependency Array-এর নিয়ম

| Dependency Array | কখন Effect চলে |
|-----------------|----------------|
| না দিলে | প্রতি render-এর পরে |
| `[]` (খালি) | শুধু প্রথম render-এর পরে (mount) |
| `[value]` | প্রথম render ও `value` পরিবর্তনে |

### API থেকে ছাত্রের ডাটা লোড করা

```jsx
import { useState, useEffect } from "react";

function StudentDashboard() {
  const [students, setStudents] = useState([]);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    // AbortController দিয়ে cleanup — component unmount হলে fetch বাতিল হবে
    const controller = new AbortController();

    async function fetchStudents() {
      try {
        setIsLoading(true);
        const response = await fetch(
          "https://api.example.com/students",
          { signal: controller.signal }
        );

        if (!response.ok) {
          throw new Error(`Server Error: ${response.status}`);
        }

        const data = await response.json();
        setStudents(data);
      } catch (err) {
        // AbortError ignore করো — এটা cleanup-এর কারণে হয়
        if (err.name !== "AbortError") {
          setError(err.message);
        }
      } finally {
        setIsLoading(false);
      }
    }

    fetchStudents();

    // Cleanup: component unmount হলে ongoing fetch বাতিল করো
    return () => controller.abort();
  }, []); // খালি array — শুধু mount-এ একবার চলবে

  if (isLoading) return <div className="text-center p-10">⏳ লোড হচ্ছে...</div>;
  if (error) return <div className="text-red-500 p-4">⚠️ {error}</div>;

  return <StudentList students={students} />;
}
```

## 14. Lifting State Up ও Context API

### Lifting State Up

যখন দুটো sibling component একই state শেয়ার করতে হয়, state-কে তাদের common parent-এ নিয়ে যেতে হয় — এটাই **Lifting State Up**।

```
App (state এখানে)
├── GradeFilter (onSelectGrade → App)
└── StudentList (filteredStudents ← App)
```

আমাদের `App.jsx`-এ আমরা ইতিমধ্যে এটা করেছি। `filterGrade` state App-এ আছে, GradeFilter পরিবর্তন করে, StudentList সেই অনুযায়ী render হয়।

### Context API — Global State

যদি state অনেক deep-এ থাকা কোনো component-কে দিতে হয়, **props drilling** (props বারবার pass করা) সমস্যা হয়। Context API এই সমস্যার সমাধান।

```jsx
// src/context/StudentContext.jsx
import { createContext, useContext, useState } from "react";
import { students as initialStudents } from "../data/students";

// Step 1: Context তৈরি করো
const StudentContext = createContext(null);

// Step 2: Provider Component তৈরি করো
export function StudentProvider({ children }) {
  const [students, setStudents] = useState(initialStudents);

  function addStudent(newStudent) {
    setStudents((prev) => [...prev, { ...newStudent, id: Date.now() }]);
  }

  function deleteStudent(id) {
    setStudents((prev) => prev.filter((s) => s.id !== id));
  }

  const value = { students, addStudent, deleteStudent };

  return (
    <StudentContext.Provider value={value}>
      {children}
    </StudentContext.Provider>
  );
}

// Step 3: Custom hook দিয়ে সহজে ব্যবহার
export function useStudents() {
  const context = useContext(StudentContext);
  if (!context) {
    throw new Error("useStudents must be used within StudentProvider");
  }
  return context;
}
```

## 16. Custom Hook — Reusable Logic

**Custom Hook** হলো এমন একটি JavaScript function যার নাম `use` দিয়ে শুরু এবং ভেতরে React Hook ব্যবহার করা হয়। এটা দিয়ে একই logic বারবার লেখা এড়ানো যায়।

### `useFetch` — API থেকে ডাটা লোড

```jsx
// src/hooks/useFetch.js
import { useState, useEffect } from "react";

export function useFetch(url) {
  const [data, setData] = useState(null);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    if (!url) return;

    const controller = new AbortController();

    async function fetchData() {
      try {
        setIsLoading(true);
        setError(null);
        const res = await fetch(url, { signal: controller.signal });
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        const json = await res.json();
        setData(json);
      } catch (err) {
        if (err.name !== "AbortError") {
          setError(err.message);
        }
      } finally {
        setIsLoading(false);
      }
    }

    fetchData();
    return () => controller.abort();
  }, [url]);

  return { data, isLoading, error };
}
```

### `useLocalStorage` — LocalStorage Sync

```jsx
// src/hooks/useLocalStorage.js
import { useState, useEffect } from "react";

export function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    try {
      const item = localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch {
      return initialValue;
    }
  });

  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);

  return [value, setValue];
}
```

## 17. Performance Optimization

### কখন optimize করবো?

> "Premature optimization is the root of all evil" — Donald Knuth

আগে কোড কাজ করাও, তারপর প্রয়োজন হলে optimize করো।

### `React.memo` — Unnecessary Re-render রোধ

কোনো Component-এর props পরিবর্তন না হলেও parent re-render হলে child-ও re-render হয়। `React.memo` এটা রোধ করে।

```jsx
// StudentCard props না বদলালে re-render হবে না
const StudentCard = React.memo(function StudentCard({ student, onDelete }) {
  console.log("Rendering:", student.name);
  // ...
});
```

### `useCallback` — Function Reference Stable রাখা

```jsx
// ❌ সমস্যা — প্রতি render-এ নতুন function তৈরি হয়
//    → React.memo কাজ করে না কারণ props পরিবর্তন হচ্ছে
function App() {
  const handleDelete = (id) => { /* ... */ }; // নতুন reference প্রতিবার!
  return <StudentList onDelete={handleDelete} />;
}

// ✅ সমাধান — useCallback দিয়ে function memoize করো
function App() {
  const handleDelete = useCallback((id) => {
    setStudents((prev) => prev.filter((s) => s.id !== id));
  }, []); // dependency নেই, একই function সবসময়

  return <StudentList onDelete={handleDelete} />;
}
```

### `useMemo` — Expensive Calculation Cache করা

```jsx
function App() {
  const [students, setStudents] = useState(initialStudents);
  const [filterGrade, setFilterGrade] = useState("all");

  // ✅ filteredStudents শুধু students বা filterGrade পরিবর্তনে recalculate হবে
  const filteredStudents = useMemo(() => {
    return filterGrade === "all"
      ? students
      : students.filter((s) => getGrade(getAverage(s)) === filterGrade);
  }, [students, filterGrade]);

  // ✅ statistics শুধু students পরিবর্তনে recalculate হবে
  const statistics = useMemo(() => {
    const total = students.length;
    const averageMarks =
      total > 0
        ? students.reduce((sum, s) => sum + getAverage(s), 0) / total
        : 0;
    const passRate =
      total > 0
        ? (students.filter((s) => getAverage(s) >= 33).length / total) * 100
        : 0;
    return { total, averageMarks, passRate };
  }, [students]);

  return (
    <>
      <Summary stats={statistics} />
      <StudentList students={filteredStudents} />
    </>
  );
}
```
